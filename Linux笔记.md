# Linux笔记



## 一、Linux基础篇



### 第一节、Linux目录结构

Linux目录结构遵循**FHS（Filesystem Hierarchy Standard，文件系统层次结构标准）**，核心是「一切皆文件」，所有目录都挂载在根目录`/`下，无Windows的盘符概念。



#### 1、基本介绍

1. 单根结构：所在目录/文件最终都归属根目录`/`（唯一顶级目录），路径以`/`开头（如`/home/milan`）；
2. 目录分类：
   - 系统目录（`/bin`/`/etc`/`/user`等）：存放系统 核心文件，仅root可修改；
   - 用户目录（`/home`）：普通用户专属，可读写；
   - 临时/设备文件（`/tmp`/`dev`）：系统运行临时文件、硬件设备映射；
3. Debian专属：部分目录（如`/var/lib/dpkg`）存放Debian包管理数据，`/etc/apt`专属APT配置，需重点关注。

---



#### 2、具体目录结构

* **`/bin`**：(`/usr/bin`、`/usr/local/bin`)【常用】

  是Binary（二进制）的缩写，这个目录存放着最经常使用的命令。

* **`/sbin`**：(`/usr/sbin`、`usr/local/bin`)

  s就是Super User的意思，这里存放的是系统管理员使用的系统管理程序。

* **`/home`**：【常用】

  存放普通用户的主目录，在Linux中每个用户都有一个自己的目录，一般该目录名是以用户的账号命名。

* **`/root`**：【常用】

  该目录为系统管理员，也称作超级权限者的用户主目录。

* **`/lib`**：

  系统开机所需要最基本的动态连接共享库，其作用类似于Windows里的DLL文件。几乎所有的应用程序都需要用到这些共享库。

* **`/lost+found`**：

  这个目录一般情况下是空的，当系统非法关机后，这里存放了一些文件。

* **`/etc`**：【常用】

  所有的系统管理所需要的配置文件和子目录。

* **`/usr`**：【常用】

  这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似于Windows下的program files目录。

* **`/boot`**：【常用】

  存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件。

* **`/proc`**：【不能动】

  这个目录是一个虚拟的目录，它是系统内存的映射，访问这个目录来获取系统信息。

* **`/srv`**：【不能动】

  service（服务）缩写，该目录存放一些服务启动之后需要提取的数据。

* **`/sys`**：【不能动】

  这是linux2.6内核的一个很大变化。该目录下安装了2.6内核中新出现的一个文件系统sysfs。

* **`/tmp`**：

  这个目录是用来存放一些临时文件的。

* **`/dev`**：

  类似于Windows的设备管理器，把所有的硬件用文件的形式存储。

* **`/media`**：【常用】

  Linux系统会自动识别一些设备，如U盘、光驱等。当识别后，Linux会把识别的设备挂载到这个目录下。

* **`/mnt`**：【常用】

  系统提供该目录是为了让用户临时挂载别的文件系统，我们可以将外部的存储挂载在`/mnt/`上，然后进入该目录就可以查看里面的内容了。

* **`/opt`**：

  这是给主机额外安装软件所摆放的目录。如安装ORACLE数据库就可放到该目录下。默认为空。

* **`/usr/local`**：【常用】

  这是另一个给主机额外安装软件所安装的目录。一般是通过编译源码方式安装的程序。

* **`/var`**：【常用】

  这个目录中存放着在不断扩张的东西，习惯将经常被修改的目录放在这个目录下。包括各种日志文件。

* **`/selinux`**：

  SElinux是一种安全子系统，它能控制程序只能访问特定文件，有三种工作模式，可以自行设置。

---



#### 3、Debian专属目录补充

- `/var/lib/dpkg`：存放Debian包管理（dpkg）的数据库，记录已安装软件的版本、文件列表，删除会导致`dpkg/apt`无法使用；

- **`/etc/default`**：Debian 服务默认配置（如 `ssh`/`ufw` 的默认参数）；

- **`/etc/network`**：Debian 网络配置目录（如 `interfaces` 配置静态 IP）；

- **`/usr/share/doc`**：Debian 已安装软件的文档（如 `sudo apt install vim` 后，`/usr/share/doc/vim` 有 vim 使用说明）；

- **`/run`**：系统运行时的临时 PID 文件、套接字文件（如 `/run/sshd.pid` 记录 SSH 进程 ID），重启后重建。

---



#### 4、实用技巧

1. 快速定位目录/文件

   ```bash
   # 查找命令所在目录（如ls）
   which ls	# 输出 /bin/ls
   
   # 查找配置文件位置（如ssh_config）
   find /etc -name "ssh_config"	# 输出 /etc/ssh/ssh_config
   
   # 查看目录大小（Debian常用，排查磁盘占用）
   du -sh /var/log	# 查看日志目录总大小（-s: 汇总，-h: 人性化显示）
   ```

2. 目录权限避坑

   - 系统目录（`/etc`/`/bin`/`/usr`）：默认仅root可写，普通用户只读，**禁止手动修改权限**；
   - 用户目录（`/home/username`）：权限必须为700（`chmod 700 /home/username`），避免其他用户访问；
   - 临时目录（`/tmp`）：权限1777（所有用户可读写，但禁止删除他人文件，系统默认配置）。

3. Debian目录结构易错点

   - 不要混淆`/homr`和`/root`：`/root`是root用户家目录，`/home`是普通用户访问；
   - 不要删除`/var/log`：日志是排查问题的关键，如需清理，用`sudo truncate -s 0 /var/log/syslog`（清空单个日志），而非删除目录；
   - `/bin`/`sbin`是软链接：Debian为简化维护，将`/bin`链接到`/usr/bin`，修改`/usr/bin`会同步影响`/bin`。

---



#### 5、总结

1. **核心目录记忆**：`/etc`（配置）、`/home`（用户）、`/var`（动态数据）、`/usr`（软件）是日常操作最多的目录；
2. **Debian 重点**：`/etc/apt`（源配置）、`/var/lib/dpkg`（包数据）、`/var/log`（日志）是排查 Debian 问题的核心；
3. **安全原则**：非 root 用户不要修改 `/` 下的系统目录，修改配置文件前先备份。

日常使用中，记住「什么文件放什么目录」即可：配置文件找 `/etc`，用户文件放 `/home`，系统命令找 `/bin`/`/sbin`，日志找 `/var/log`，基本能覆盖 90% 的操作场景。

## 二、Linux实操篇



### 第一节、远程登陆到Linux服务器

Windows系统上使用XShell软件远程登陆，Mac系统自带ssh服务，使用终端连接。

Mac终端可以直接使用自带的SSH功能连接Linux系统，无需额外安装客户端。



#### 1、前期准备

- 确认Linux端已开启SSH服务：登陆Linux系统，执行命令`sudo systemctl status sshd`，若显示“active (running)”，则说明服务已启动；若未启动，执行`sudo systemctl start sshd`开启，且可通过`sudo systemctl enable sshd`设置开机自启。

- 收集关键信息：记下Linux系统的**IP地址**（可通过`ifconfig`或`ip addr`命令查看）、**登录用户名**（如root、自定义普通用户名），以及对应登录密码。

- 开放SSH端口：Linux端若开启防火墙，需放行22号端口（SSH默认端口）。

---



#### 2、方式一：密码连接（快速上手）

1. 在Mac终端打开终端。
2. 输入SSH连接命令：终端中输入基础命令`ssh username@LinuxIP`。例如Linux用户名为root，IP为192.168.0.1，命令即为`ssh root@192.168.0.1`；若Linux修改了SSH默认端口（如2222），需要`-p`指定端口，命令改为`ssh -p 2222 root@192.168.0.1`。
3. 验证并输入密码，首次连接会弹出服务器指纹验证提示，输入`yes`回车确认。接着输入Linux用户名对应的密码，注意输入时终端不会显示任何字符输入完成后回车即可。
4. 连接成功与退出：若密码正确，终端提示符会变为Linux端的样式（如root@linux:~#），此时就能输入Linux命令操作了；想断开连接时，输入`exit`或按下`Ctrl+D`即可。

---



#### 3、方式二：密钥连接（免密且更安全）

1. Mac端生成密钥对：在终端输入`ssh-keygen -t ed25519`（ed25519算法更安全），之后连续回车即可，默认会在`~/.ssh/`目录下生成私钥`id_ed25519`和公钥`id_ed25519.pub`。
2. 上传公钥到Linux：终端执行命令`ssh-copy-id -i ~/.ssh/id_25519.pub username@LinuxIP`，比如`ssh-copy-id -i ~/.ssh/id_ed25519.pub root@192.168.0.1`，输入Linux密码完成公钥上传。
3. 直接连接Linux：后续只需要在Mac终端输入`ssh username@LinuxIP`，无需输入密码就能自动连接成功。若私钥保存路径非默认，连接时用`-i`指定路径，命令如`ssh -i ~/Documents/id_ed25519 root@192.168.0.1`。

### 第二节、vi和vim编辑器



#### 1、基本介绍

Vi是Unix/Linux系统内置的基础文本编辑器，Vim（Vi Improved）是Vi的增强版（Debian默认预装Vim），相比Vi增加了语法高亮、代码补全、多窗口编辑等实用功能，是Linux下修改配置文件、编写核心代码的核心工具。

---



#### 2、Vim三种核心模式

Vim核心特点是「模式化编辑」，不同模式对应不同操作。

| 模式名称            | 核心功能                             | 切换方式                               | 典型操作场景                        |
| ------------------- | ------------------------------------ | -------------------------------------- | ----------------------------------- |
| 普通模式（Normal）  | 快捷键操作（移动、复制、删除、撤销） | 打开Vim默认进入；其他模式下按`Esc`切换 | 光标定位、批量删除/复制、执行快捷键 |
| 插入模式（Insert）  | 输入/编辑文本                        | 普通模式按`i/a/o`等键切换              | 写代码、修改配置文件内容            |
| 命令模式（Command） | 执行保存、退出、搜索、替换等命令     | 普通模式按`:`、`/`、`?`切换            | 保存文件、退出Vim、全局替换文本     |

新手避坑：

- 打开文件后先按`Esc`确认进入普通模式，再执行快捷键；
- 输入文字前必须进入插入模式，否则按键会触发快捷键（比如按`x`会删除字符而非输入）。

---



#### 3、Vim基础操作

1. 启动与退出

   ```bash
   # 1.打开/创建文件
   vim /etc/apt/sources.list		# 编辑APT源配置（需sudo提权）
   sudo vim /etc/ssh/sshd_config		# 编辑SSH配置（需root权限）
   vim test.txt	# 打开当前目录的test.txt，不存在则创建
   
   # 2.退出Vim
   # 场景1:未修改内容，直接退出
   Exc -> :q -> 回车
   
   # 场景2:修改后保存并退出
   Exc -> :wq -> 回车（或Exc -> :x -> 回车）
   
   # 场景3:修改后放弃更改，强制退出
   Exc -> :q! -> 回车
   
   # 场景4:保存但不退出（编辑中临时保存）
   Exc -> :w -> 回车
   ```

   <br>

2. 模式切换与光标移动（普通模式）

   （1）模式切换快捷键

   | 快捷键 | 功能                       | 适用场景                           |
   | ------ | -------------------------- | ---------------------------------- |
   | `i`    | 在光标前插入文本           | 光标在文字中间时编辑               |
   | `a`    | 在光标后插入文本           | 避免覆盖当前字符                   |
   | `o`    | 在光标下一行新建一行并插入 | 换行编辑，排版更整洁               |
   | `Esc`  | 退出插入模式进入普通模式   | 输入完文字后切回普通模式执行快捷键 |

   （2）光标移动（普通模式，比方向键高效）

   | 快捷键    | 功能                 | 进阶技巧                  |
   | --------- | -------------------- | ------------------------- |
   | `h/j/k/l` | 左/下/上/右移动      | 配合数字：`5j`向下移动5行 |
   | `gg`      | 跳转到文件首行       | 快速定位配置文件开头      |
   | `G`       | 跳转到文件尾行       | 查看配置文件最后内容      |
   | `nG`      | 跳转到第n行          | `10G`跳转到第10行         |
   | `w`       | 跳转到下一个单词开头 | 快速跳过整段文字          |
   | `b`       | 跳转到上一个单词开头 | 回退查看前一个关键词      |

   （3）文本编辑（普通模式，高频操作）

   | 快捷键      | 功能                  | 示例（Debian配置文件修改）                    |
   | ----------- | --------------------- | --------------------------------------------- |
   | `yy`        | 复制当前行            | 修改SSH端口时，先`yy`复制原配置行，再修改备用 |
   | `nyy`       | 复制从当前行开始的n行 | `3yy`复制3行                                  |
   | `dd`        | 删除当前行（可粘贴）  | 删除无用的配置行                              |
   | `ndd`       | 删除从当前行开始的n行 | `2dd`删除2行                                  |
   | `p`         | 粘贴到光标下一行      | 复制配置行后，`p`粘贴到目标位置               |
   | `P`（大写） | 粘贴到光标上一行      | 插入到目标行上方                              |
   | `x`         | 删除光标所在的字符    | 修正单个输错的字符                            |
   | `u`         | 撤销上一步操作        | 误删/误改时                                   |
   | `Ctrl + r`  | 恢复撤销的操作        | 撤销错了再恢复                                |

   （4）搜索与替换（命令模式，Debian改配置必备）

   1️⃣快速搜索

   ```bash
   # 普通模式按/进入搜索，输入关键词后回车
   /Port		# 搜索“Port”（如找SSH配置中的端口行）
   n				# 跳转到下一个匹配结果
   N				# 跳转到上一个匹配结果
   ```

   2️⃣文本替换

   ```bash
   # 命令模式执行，格式：:范围 s/旧内容/新内容/选项
   :1,$ s/22/2222/g	# 全局替换：将1到最后一行的22改为2222（改SSH端口）
   :% s/old/new/g		# 等价于上一行，%代表所有行，g代表全局匹配
   :5,10 s/test/pord/	# 局部替换：仅5-10行的test改为pord（不该其他行）
   ```

   - 关键参数：`g`（全局替换，否则仅替换没行第一个匹配项）、`c`（替换前确认，`:1,$ s/22/2222/gc`）。

---



#### 4、Vim进阶技巧

1. 显示行号

   ```bash
   # 临时显示行号（仅当前会话有效）
   Esc -> :set nu -> 回车
   
   # 临时隐藏行号
   Esc -> :set nonu -> 回车
   
   # Debian永久显示行号
   # 编辑Vim配置文件（无则创建）
   vim ~/.vimrc
   # 加入一行：set nu
   # 保存退出后，下次打开Vim自动显示行号
   ```

   <br>

2. 多窗口编辑

   ```bash
   # 同时打开两个文件，左右分屏
   vim -o /etc/ssh/sshd_config /etc/ssh/sshd_config
   
   # 窗口切换：普通模式按Ctrl + w -> 再按l（右）/h（左）
   ```

   <br>

3. 批量缩进（编写脚本/配置文件）

   ```bash
   # 普通模式下，选中多行（按 v 进入可视模式，上下移动选中）
   # 缩进：按 >（向右缩进）/ <（向左缩进）
   # 快速缩进当前行：>>（右）/ <<（左）
   ```

   <br>

4. 读取系统命令输出

   ```bashe
   # 命令模式下，读取ls命令的输出并插入到文件中
   :r !ls /etc/apt	# 将/etc/apt目录下的文件列表插入到光标位置
   ```

---



#### 5、Debian专属配置与避坑

1. Debian中Vim的安装与默认配置

   ```bash
   # 检查是否安装Vim
   vim --version		# 无输出则安装
   sudo apt update && sudo apt install vim -y
   
   # Debian默认vim配置文件位置
   /etc/vim/vimrc	# 系统全局配置（修改需root）
   ~/.vimrc	# 用户个人配置（优先级更高，推荐修改这个）
   ```

   <br>

2. 新手常见坑（Debian场景）

   - 权限问题：编辑系统配置文件（如`/etc/apt/sources.list`）时，未加`sudo`导致无法保存，需退出后重新执行`sudo vim 文件名`；

   - 误删配置：修改Debian核心配置（如`/etc/fstab`）前，先复制备份，再用Vim编辑；

   - 退出失败：插入模式下直接按`:q`会输入文字而非退出，需要先按`Esc`回到普通模式；

   - 中文乱码：Debian下编辑中文文件时，在`~/.vimrc`加入一下内容：

     ```bash
     set fileencodings=utf-8,gb2312,gbk,gb18030
     set termencoding=utf-8
     set encoding=utf-8
     ```

---



#### 6、总结

1. **模式切换**：打开 Vim 先按 `Esc` 到普通模式，输入文字按 `i/a/o`，执行指令按 `:`；
2. **核心退出**：`:wq`（保存退出）、`:q!`（强制退出）、`:w`（保存）；
3. **高频操作**：`yy`（复制）、`dd`（删除）、`p`（粘贴）、`u`（撤销）、`/关键词`（搜索）；
4. **Debian 技巧**：编辑系统文件加 `sudo`，修改配置前备份，永久显示行号改 `~/.vimrc`。

日常使用中，记住「模式切换 + 保存退出 + 基础编辑」就能覆盖 Debian 下 90% 的 Vim 操作场景，进阶功能（如多窗口、替换）可按需学习，不用一次性记完。

### 第三节、关机、重启和用户注销

Linux的关机、重启与用户注销均需区分权限与场景，Debian默认采用systemd管理系统，同时兼容传统命令。



#### 1、核心前提：权限与安全原则

- 关机/重启：仅root或拥有sudo权限的用户可执行（普通用户无权限，避免误操作）；
- 安全优先：执行关机/重启前，需保存所有工作、停止关键服务（如MySQL、Nginx），避免数据丢失或文件系统损坏；
- Debian默认：桌面版支持图形化操作（如右上角电源菜单），服务器版推荐命令行操作，更可控。

---



#### 2、关机操作

关机的核心是「通知进程➡️同步数据➡️卸载文件系统➡️切断电源」，以下是常用命令与场景：

1. 推荐命令：shutdown（最安全，支持延迟与通知）

   ```bash
   # 基本语法
   shutdown [选项] [时间] [提示消息]
   ```

   | 命令                                          | 功能                                                  | 场景           |
   | --------------------------------------------- | ----------------------------------------------------- | -------------- |
   | sudo shutdown -h now                          | 立即关机并切断电源                                    | 日常快速关机   |
   | sudo shutdown -h +5 "5分钟后关机，请保存工作" | 5分钟后关机，向所有登录用户发通知                     | 服务器维护提醒 |
   | sudo shutdown -h 22:00                        | 22:00准时关机                                         | 定时维护       |
   | sudo shutdown -c                              | 取消以计划的关机                                      | 临时取消维护   |
   | sudo shutdown -P now                          | 明确关机并切断电源（等价于 -h now，Debian中效果一致） | 确保电源关闭   |

   <br>

2. 快捷命令（适合本地操作）

   | 命令                    | 说明                                    | Debian适配                           |
   | ----------------------- | --------------------------------------- | ------------------------------------ |
   | sudo poweroff           | 立即关机（等价于 sudo shutdown -h now） | 兼容 systemd，默认切断电源           |
   | sudo halt               | 停止系统（不切断电源，需手动断电）      | 仅用于无 systemd 的就系统，不推荐    |
   | sudo systemctl poweroff | systemd 原生关机命令                    | Debian 9+ 默认优先，与 poweroff 等价 |

---



#### 3、重启操作

与关机逻辑一致，仅最终步骤为重启。

1. 推荐命令：shutdown -r（安全重启）

   ```bash
   sudo shutdown -r now		# 立即重启
   sudo shutdown -r +10		# 十分钟后重启，发送通知
   sudo shutdown -r 3:00		# 定时3:00重启
   ```

   <br>

2. 快捷命令

   | 命令                  | 说明                                     | 应急场景               |
   | --------------------- | ---------------------------------------- | ---------------------- |
   | sudo reboot           | 立即重启（等价于sudo shutdown -r now）   | 日常快速重启           |
   | sudo systemctl reboot | systemd 原生重启命令                     | Debian推荐             |
   | sudo reboot -f        | 强制重启（跳过正常关机流程，有数据风险） | 系统卡死无法重启时使用 |
   | sudo init 6           | 通过运行级别重启（传统方式，Debian兼容） | 无 systemd 的旧环境    |

#### 4、用户注销

注销仅影响但前用户会话，不影响系统运行，以下是常用方式：

1. 终端注销（最常用）

   | 操作     | 功能                                   | 适用场景                            |
   | -------- | -------------------------------------- | ----------------------------------- |
   | logout   | 注销当前登录Shell（仅对登录Shell有效） | SSH/控制台登录后退出                |
   | exit     | 退出当前Shell（非登录Shell也可用）     | su 切换用户后返回上一级，或关闭终端 |
   | Ctrl + D | 快捷键，等价于 exit                    | 快速退出终端                        |

   <br>

2. 注销其他用户（root权限）

   ```bash
   # 查看当前登录用户
   w		# 显示用户名、终端、登录时间
   # 终止指定用户的会话（如用户 milan 的终端 pts/0）
   sudo pkill -u milan	# 终止该用户的所有进程（谨慎使用）
   sudo pkill -9 <pid>	# 精准终止单个会话进程（推荐，先通过 ps 查 pid）
   ```

   <br>

3. Debian图形化注销

   桌面版（如Debian GNOME）可通过右上角菜单点击「注销」，或快捷键`Ctrl+Alt+Delete`触发注销选项。

---



#### 5、Debian专属特性与避坑要点

1. 系统默认配置

   - Debian 9 + 默认使用 systemd，`systemctl poweroff/reboot` 与传统命令等价；

   - 服务器版默认禁止 root 远程登录，需通过 `sudo` 执行关机 / 重启；

   - 日志记录：关机 / 重启记录会写入 `/var/log/syslog`，可通过 `grep shutdown /var/log/syslog` 查看历史操作。

2. 新手常见坑

   - **权限不足**：普通用户执行关机命令会报错，需加 `sudo`（如 `sudo shutdown -h now`）；

   - **误操作强制重启**：`reboot -f` 会跳过数据同步，可能导致文件损坏，仅用于应急；

   - **注销与关机混淆**：`logout` 仅退出当前用户，不会关机，避免误操作；

   - **计划任务冲突**：使用 `shutdown` 延迟操作时，若需取消，必须用 `shutdown -c`，不可直接关闭终端。

3. 安全操作流程（Debian 服务器必看）

   1️⃣ 用 `w` 或 `who` 查看在线用户，提醒保存工作；

   2️⃣ 停止关键服务（如 `sudo systemctl stop mysql`）；

   3️⃣ 执行同步命令（`sync`，确保数据写入磁盘）；

   4️⃣ 执行关机 / 重启命令（优先用 `shutdown` 或 `systemctl`）。

---



#### 6、总结（核心命令速记）

| 操作         | 推荐命令               | 应急命令      |
| ------------ | ---------------------- | ------------- |
| 立即关机     | sudo shutdown -h now   | sudo poweroff |
| 立即重启     | sudo shutdown -r now   | sudo reboot   |
| 注销当前用户 | logout / exit / Ctrl+D | -             |
| 取消计划操作 | sudo shutdown -c       | -             |

日常使用中，优先选择 `shutdown` 或 `systemctl` 系列命令，兼顾安全性与兼容性；用户注销用 `logout` 或 `Ctrl+D` 即可，简单高效。
